Upload a GSC export to find and forecast high-ROI keywords.

How to get your export data (3 Methods)
Recommended

Google Sheets Add-on

  1. Install the free Search Analytics for Sheets extension.
  2. Open a blank Google Sheet and launch the extension.
  3. Select your website and desired date range.
  4. Under "Group By", select: Query and Page.
  5. Click "Request Data", download as CSV, and upload here.
Advanced

Looker Studio Dashboard

  1. Create a blank report in Looker Studio.
  2. Add Data -> Google Search Console -> URL Impression -> Web.
  3. Insert a Table chart.
  4. Dimensions: Query and Landing Page.
  5. Metrics: Impressions, Url Clicks, and Average Position.
  6. Click three dots -> Export -> CSV.
Quickest

Standard GSC Export

  1. Log into standard Google Search Console.
  2. Go to Performance and set your date range.
  3. Click "Export" -> Download CSV.
  4. Upload ONLY the Queries.csv file here.

CTR & Target Position

The default 10% CTR at position 3 is a generic estimate. Set your own target rank and expected click-through rate — a B2B software query and a local plumbing query convert very differently at the same position.

Commonly cited averages (vary a lot by vertical)

Pos 1 ≈ 25–30% · Pos 2 ≈ 12–18% · Pos 3 ≈ 8–12% · Pos 5 ≈ 4–6% · Pos 10 ≈ 1–3%. Treat these as a rough starting point, not your actual rate — pull your own CTR from GSC for your best-performing pages if you can.

What does the Striking Distance Analyzer do?

This tool finds "striking distance" keywords — search terms your site already ranks for on page 2 or 3 of Google (roughly positions 11–30) with meaningful impression volume — from a Google Search Console export. These are usually the fastest wins available in SEO: a page already has some relevance and authority for the term, so a moderate on-page or content push can be enough to move it onto page 1, rather than needing months of link building from a standing start.

Upload your GSC export (three methods are covered in the instructions panel: the Search Analytics for Sheets add-on, a Looker Studio dashboard, or the standard GSC export), filter by position range, impressions, keyword, landing page, or brand terms, and the tool ranks every keyword by an Opportunity Score and forecasts the traffic uplift against a target position and CTR you control. Switch to Page View to see the same data rolled up by URL — the content-brief view for deciding which page to rewrite first — and watch for the cannibalization warning if the same keyword is splitting rankings across more than one page. Export the prioritised list as a CSV to work from directly.

Frequently asked questions

Is the Striking Distance Analyzer free to use?

Yes, completely free with no account or usage limit.

What is a "striking distance" keyword?

It's a keyword your page already ranks for, but just outside page 1 — commonly defined as positions 11 to 30 — where a page already has enough relevance and authority that pushing it further usually takes less work than ranking a page from scratch. They're typically the highest-ROI keywords to prioritise in an SEO roadmap.

Is my Search Console data uploaded anywhere?

No, it's never sent to a server — everything is parsed and analysed locally using PapaParse. It is, however, saved in your browser's local storage on this device so your CSV, filters, and settings are still there if you close the tab or come back later. Use the "Clear Saved Data" button any time to remove it.

Why do I need to enter my domain for some exports?

The standard GSC export (Queries.csv) doesn't include landing page URLs, only queries. When that's detected, the tool asks for your domain so it can generate a site: Google search link for each keyword instead — letting you quickly find which page is ranking, since the export itself doesn't tell you. Page View and cannibalization detection both need real landing page data, so they're unavailable for a domain-only standard export.

How is the Opportunity Score calculated?

It's (31 − average position) × impressions — weighting keywords that are both closer to page 1 and have higher search volume more heavily, so the highest-scoring rows at the top of the table represent the best combination of "close to ranking" and "worth ranking for." In Page View, a page's score is the sum of the Opportunity Scores of every striking-distance keyword currently matching it.

Is the CTR assumption accurate for my site?

By default it's 10% at position 3, a commonly cited industry rule-of-thumb, not a guarantee — actual click-through rate varies a lot by query intent, SERP features present, and your title/meta description. Open "CTR Settings" to set your own target position and CTR based on your own site's data if you have it; the forecast is directional either way, useful for prioritisation rather than a precise prediction.

What does the cannibalization warning mean?

It means the same keyword, within your current filters, is showing up ranking for more than one landing page on your site — which typically splits ranking signals across both pages and keeps either from reaching page 1. It's usually worth consolidating the weaker page into the stronger one with a 301 redirect, or clearly differentiating the two pages' targeting, rather than optimising them separately.

How does Page View differ from Keyword View?

Keyword View lists every striking-distance keyword as its own row. Page View groups those same keywords by landing page and sums their Opportunity Scores into a single Page Opportunity Score, so you can see at a glance which article or page has the most striking-distance keywords worth targeting — expand a page's row to see the individual keywords behind its score.
